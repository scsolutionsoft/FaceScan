# Student Care Module Design

## เป้าหมาย

เพิ่มโมดูล "ระบบช่วยเหลือดูแลนักเรียน" ให้ FaceScan โดยต่อยอดจากข้อมูลนักเรียน, บัญชีนักเรียน, บทบาทครู, ห้องเรียน และระบบรายงานที่มีอยู่แล้ว ระบบใหม่นี้ต้องช่วยโรงเรียนติดตามพฤติกรรม ความดี การเยี่ยมบ้าน ฐานะครอบครัว ความเสี่ยง และกิจกรรมธนาคารขยะได้ในที่เดียว

## โมดูลย่อย

ระบบแบ่งเป็น 5 โมดูลย่อย และทุกโมดูลต้องเปิดปิดได้จากหน้า Settings

1. คะแนนพฤติกรรม
2. ธนาคารความดี
3. ระบบเยี่ยมบ้าน
4. ธนาคารขยะ
5. รายงานช่วยเหลือดูแลนักเรียน

ค่าการเปิดปิดควรเพิ่มใน `SystemSetting`

- `EnableStudentCareModule`
- `EnableBehaviorScoreModule`
- `EnableGoodnessBankModule`
- `EnableHomeVisitModule`
- `EnableWasteBankModule`

เมื่อปิดโมดูล เมนูและ action ที่เกี่ยวข้องต้องถูกซ่อน และ controller ต้องตรวจสอบซ้ำฝั่ง server เพื่อป้องกันการเรียก URL โดยตรง

## บทบาทผู้ใช้งาน

### นักเรียน

- เข้าสู่ระบบผ่าน Student Portal เดิม
- ตรวจสอบคะแนนพฤติกรรมของตนเอง
- ตรวจสอบคะแนนธนาคารความดีของตนเอง
- ตรวจสอบยอดเงินหรือประวัติธุรกรรมธนาคารขยะของตนเอง
- แชร์พิกัดบ้านจากโทรศัพท์มือถือ โดยใช้ browser geolocation
- ยืนยันหรือปรับปรุงข้อมูลติดต่อผู้ปกครองบางส่วนตามสิทธิ์ที่โรงเรียนกำหนด

### ครูที่ปรึกษา

- ดูรายชื่อนักเรียนในห้องที่รับผิดชอบ
- บันทึกคะแนนพฤติกรรม เพิ่ม/ลด พร้อมเหตุผลและหลักฐาน
- บันทึกความดีหรือกิจกรรมจิตอาสา
- สร้างใบนัดเยี่ยมบ้านและบันทึกผลเยี่ยมบ้าน
- ถ่ายภาพบ้านและภาพผู้ปกครองตอนเยี่ยมบ้านผ่านมือถือ
- บันทึกข้อมูลนักเรียน เช่น อาศัยอยู่กับใคร รายได้ครัวเรือน พฤติกรรมเสี่ยง ปัญหาที่พบ และข้อเสนอแนะ
- ส่งแบบฟอร์มรายงานให้หัวหน้าระดับ/ฝ่ายปกครองตรวจสอบ

### งานปกครอง/แอดมิน

- ตั้งค่ากติกาคะแนนพฤติกรรม
- ตั้งค่าประเภทความดีและคะแนนที่ได้รับ
- ตั้งค่าราคารับซื้อขยะต่อกิโลกรัม
- ตรวจสอบและแก้ไขข้อมูลที่ครูส่ง
- เปิดปิดโมดูล
- ส่งออกรายงาน Excel/PDF

### ผู้บริหาร

- ดูแดชบอร์ดภาพรวมระดับโรงเรียน
- ดูรายงานความเสี่ยงตามชั้น ห้อง เพศ ระดับรายได้ และสถานะการเยี่ยมบ้าน
- ดูแนวโน้มคะแนนพฤติกรรม ความดี และธนาคารขยะ

### WasteBankTransaction

เก็บรายการฝากขยะและแปลงเป็นเงิน

- `StudentId`
- `AcademicYearId`
- `WasteType`
- `WeightKg`
- `PricePerKg`
- `Amount`
- `RecordedByUserId`
- `RecordedAt`
- `Note`

สูตรคำนวณพื้นฐาน:

```text
Amount = WeightKg * PricePerKg
```

ตัวอย่าง:

```text
ขวดพลาสติกใส 2.50 กิโลกรัม
ราคากิโลกรัมละ 8.00 บาท
ยอดเงิน = 2.50 * 8.00 = 20.00 บาท
```

ระบบควรบันทึก `PricePerKg` ลงใน transaction ทันที เพื่อให้รายงานย้อนหลังไม่เปลี่ยนเมื่อมีการปรับราคาในอนาคต

## Workflow หลัก

### คะแนนพฤติกรรม

1. แอดมินตั้งคะแนนตั้งต้น เช่น 100 คะแนนต่อปีการศึกษา
2. ครูหรือฝ่ายปกครองบันทึกเหตุการณ์เพิ่ม/ลดคะแนน
3. ระบบคำนวณคะแนนคงเหลือแบบ real-time
4. นักเรียนดูคะแนนและประวัติได้ใน Student Portal
5. ผู้บริหารดูรายงานนักเรียนกลุ่มเสี่ยงจากคะแนนต่ำกว่าค่าที่กำหนด

### ธนาคารความดี

1. แอดมินกำหนดประเภทความดี เช่น ช่วยงานโรงเรียน, จิตอาสา, เก็บของคืน
2. ครูบันทึกความดีพร้อมคะแนนและหลักฐาน
3. ระบบรวมคะแนนตามปีการศึกษา ห้องเรียน และนักเรียน
4. นักเรียนดูคะแนนสะสมของตนเอง
5. โรงเรียนใช้รายงานเพื่อยกย่องหรือมอบเกียรติบัตร

### เยี่ยมบ้าน

1. ครูที่ปรึกษาเปิดรายชื่อนักเรียนในห้อง
2. นักเรียนแชร์พิกัดบ้านจากมือถือ หรือครูบันทึกพิกัดขณะเยี่ยมบ้าน
3. ครูกรอกแบบฟอร์มเยี่ยมบ้าน
4. ครูถ่ายภาพบ้านและภาพผู้ปกครองผ่านมือถือ
5. ครูระบุสภาพครอบครัว รายได้ นักเรียนอาศัยอยู่กับใคร พฤติกรรมเสี่ยง และปัญหาที่พบ
6. ระบบสร้างรายงานรายคน รายห้อง และภาพรวมโรงเรียน

### ธนาคารขยะ

1. แอดมินตั้งชนิดขยะและราคาต่อกิโลกรัม
2. เจ้าหน้าที่หรือครูชั่งน้ำหนักขยะของนักเรียน
3. ระบบดึงราคาปัจจุบันและคำนวณยอดเงิน
4. บันทึกยอดเงินเป็นบัญชีธนาคารขยะของนักเรียน
5. นักเรียนดูประวัติรายการและยอดสะสมของตนเอง
6. ผู้บริหารดูรายงานปริมาณขยะ เงินสะสม และผลลัพธ์ด้านสิ่งแวดล้อม

- ข้อมูลนักเรียน
- ข้อมูลผู้ปกครองหลัก
- นักเรียนอาศัยอยู่กับใคร
- จำนวนสมาชิกในครอบครัว
- รายได้ครัวเรือนต่อเดือน
- อาชีพผู้ปกครอง
- สภาพบ้านและสภาพแวดล้อม
- ความสัมพันธ์ในครอบครัว
- การสนับสนุนการเรียนที่บ้าน
- พฤติกรรมเสี่ยง เช่น ขาดเรียนบ่อย, มาสายบ่อย, ใช้สารเสพติด, ติดเกม, ความรุนแรง, ภาวะซึมเศร้า
- ปัญหาที่พบ
- สิ่งที่โรงเรียนควรช่วยเหลือ
- แผนติดตามผล
- ภาพถ่ายบ้าน
- ภาพถ่ายผู้ปกครอง
- พิกัดบ้าน
- ลายเซ็นหรือการยืนยันของครูผู้เยี่ยม

- นักเรียนที่ยังไม่ได้เยี่ยมบ้าน
- นักเรียนที่มีความเสี่ยงระดับสูง
- รายงานเด็กพฤติกรรมเสี่ยง
- นักเรียนครัวเรือนรายได้น้อย
- สรุปนักเรียนอาศัยอยู่กับใคร
- สรุปรายได้ครัวเรือนตามห้อง/ระดับชั้น
- สรุปธนาคารขยะตามนักเรียน ห้อง ระดับชั้น และช่วงเวลา
- สรุปเงินธนาคารขยะคงเหลือ
- สรุปปริมาณขยะพลาสติกที่รวบรวมได้

## รายงานเด็กพฤติกรรมเสี่ยง

รายงานนี้ใช้สำหรับครูที่ปรึกษา ฝ่ายปกครอง และผู้บริหาร เพื่อติดตามนักเรียนที่ต้องได้รับการดูแลเป็นพิเศษ โดยรวบรวมข้อมูลจากคะแนนพฤติกรรม การมาเรียน การเยี่ยมบ้าน และข้อมูลที่ครูบันทึกในระบบช่วยเหลือนักเรียน

### เกณฑ์คัดกรองความเสี่ยง

ระบบควรให้แอดมินตั้งค่าเกณฑ์ได้ และมีค่าเริ่มต้นดังนี้

- คะแนนพฤติกรรมต่ำกว่าเกณฑ์ เช่น ต่ำกว่า 60 คะแนน
- ถูกตัดคะแนนพฤติกรรมซ้ำในช่วง 30 วัน
- ขาดเรียนหรือมาสายบ่อยจากข้อมูล FaceScan
- มีพฤติกรรมเสี่ยงจากแบบฟอร์มเยี่ยมบ้าน เช่น ติดเกม ใช้สารเสพติด ความรุนแรง ภาวะซึมเศร้า หรือออกนอกบ้านกลางคืน
- มีปัญหาครอบครัวหรือรายได้ครัวเรือนต่ำ
- ยังไม่ได้รับการเยี่ยมบ้านภายในช่วงเวลาที่กำหนด
- ครูที่ปรึกษาระบุให้ติดตามเป็นกรณีพิเศษ

### ระดับความเสี่ยง

- ปกติ: ไม่มีสัญญาณเสี่ยงสำคัญ
- เฝ้าระวัง: มีสัญญาณเสี่ยง 1 ด้าน หรือคะแนนเริ่มลดลง
- เสี่ยง: มีสัญญาณเสี่ยงหลายด้าน หรือคะแนนต่ำกว่าเกณฑ์
- เร่งด่วน: มีพฤติกรรมรุนแรง ปัญหาความปลอดภัย หรือควรส่งต่อฝ่ายปกครองทันที

### ข้อมูลที่แสดงในรายงาน

- รหัสนักเรียน
- ชื่อนักเรียน
- ระดับชั้น/ห้อง
- ครูที่ปรึกษา
- คะแนนพฤติกรรมคงเหลือ
- จำนวนครั้งที่ถูกตัดคะแนน
- จำนวนวันขาดเรียน/มาสาย
- สถานะเยี่ยมบ้าน
- นักเรียนอาศัยอยู่กับใคร
- รายได้ครัวเรือนหรือช่วงรายได้
- พฤติกรรมเสี่ยงที่พบ
- ปัญหาที่พบ
- ระดับความเสี่ยง
- วันที่ติดตามล่าสุด
- แผนช่วยเหลือหรือข้อเสนอแนะ

### ตัวกรองรายงาน

- ปีการศึกษา
- ระดับชั้น
- ห้องเรียน
- ครูที่ปรึกษา
- ระดับความเสี่ยง
- ประเภทพฤติกรรมเสี่ยง
- ช่วงคะแนนพฤติกรรม
- สถานะเยี่ยมบ้าน
- ช่วงวันที่บันทึกข้อมูล

### การทำงานที่ควรมี

- คลิกชื่อนักเรียนเพื่อเปิดแฟ้มดูแลรายบุคคล
- แสดง badge สีตามระดับความเสี่ยง
- export เป็น Excel/PDF
- สรุปจำนวนเด็กเสี่ยงตามระดับชั้นและห้อง
- แสดงรายการที่ต้องติดตามต่อ เช่น ยังไม่ได้เยี่ยมบ้าน, ยังไม่มีแผนช่วยเหลือ, หรือไม่มีการอัปเดตเกิน 30 วัน
- จำกัดสิทธิ์ข้อมูลอ่อนไหว โดยครูเห็นเฉพาะห้องตนเอง และผู้บริหารเห็นภาพรวมตามบทบาท

## การเชื่อมกับระบบเดิม

- ใช้ `Student` เป็นแกนกลางของข้อมูลทั้งหมด
- ใช้ `ApplicationUser.StudentId` สำหรับ Student Portal
- ใช้ `ApplicationUser.AssignedClassroomId` เพื่อจำกัดครูที่ปรึกษาให้เห็นเฉพาะห้องของตน
- ใช้ `AcademicYear` แยกข้อมูลรายปีการศึกษา
- ใช้ `IFileStorageService` เพิ่มเมธอดสำหรับเก็บรูปบ้าน, รูปผู้ปกครอง, รูปหลักฐานพฤติกรรม และรูปหลักฐานความดี
- ใช้ `AuditLog` บันทึกการแก้ไขข้อมูลอ่อนไหว
- ใช้ `Executive/Reports` เดิมเป็นที่รวมรายงานภาพรวม

## ความปลอดภัยและข้อมูลส่วนบุคคล

ข้อมูลเยี่ยมบ้าน รายได้ ภาพผู้ปกครอง ภาพบ้าน และพิกัดบ้านเป็นข้อมูลอ่อนไหว ต้องออกแบบด้วยหลักการจำกัดสิทธิ์

- นักเรียนเห็นเฉพาะข้อมูลของตนเอง
- ครูที่ปรึกษาเห็นเฉพาะนักเรียนในห้องที่ได้รับมอบหมาย
- แอดมินและผู้บริหารเห็นตามบทบาท
- ต้องมี audit log เมื่อดูหรือแก้ไขข้อมูลอ่อนไหว
- รูปภาพควรเก็บใน path แยก เช่น `uploads/student-care`
- หลีกเลี่ยงการแสดงพิกัดแบบเปิดเผยในรายงานสาธารณะ
- เพิ่ม consent text ก่อนนักเรียนแชร์พิกัดบ้าน

## แผนพัฒนาเป็นเฟส

### Phase 1: Foundation และ Toggle โมดูล

เป้าหมาย: เตรียมฐานข้อมูลและระบบเปิดปิดโมดูล

ขอบเขต:

- เพิ่ม setting เปิดปิดโมดูล
- เพิ่มเมนูตามสิทธิ์และสถานะโมดูล
- เพิ่ม entity หลักของระบบช่วยเหลือนักเรียน
- เพิ่ม migration
- เพิ่มหน้าแดชบอร์ดเริ่มต้นของครูที่ปรึกษา

ผลลัพธ์ที่ส่งมอบ:

- แอดมินเปิดปิดโมดูลได้
- ครูเห็นเมนูดูแลนักเรียนเมื่อเปิดโมดูล
- ระบบมีฐานข้อมูลพร้อมรองรับข้อมูลจริง

### Phase 2: คะแนนพฤติกรรมและธนาคารความดี

เป้าหมาย: ให้นักเรียนตรวจสอบคะแนน และให้ครูบันทึกเหตุการณ์ได้

ขอบเขต:

- ฟอร์มบันทึกเพิ่ม/ลดคะแนนพฤติกรรม
- ฟอร์มบันทึกความดี
- Student Portal แสดงคะแนนทั้งสองส่วน
- รายงานคะแนนต่ำและคะแนนความดีสูง
- export CSV/Excel

ผลลัพธ์ที่ส่งมอบ:

- นักเรียนเห็นคะแนนของตนเอง
- ครูบันทึกข้อมูลได้
- ผู้บริหารดูรายงานเบื้องต้นได้

### Phase 3: ระบบเยี่ยมบ้าน

เป้าหมาย: เก็บข้อมูลครอบครัว ภาพถ่าย และพิกัดบ้านอย่างเป็นระบบ

ขอบเขต:

- นักเรียนแชร์พิกัดบ้านผ่านมือถือ
- ครูกรอกแบบฟอร์มเยี่ยมบ้าน
- ครูถ่ายรูปบ้านและผู้ปกครอง
- บันทึกข้อมูลผู้ปกครอง รายได้ และปัญหาที่พบ
- รายงานสถานะเยี่ยมบ้านและนักเรียนกลุ่มเสี่ยง

ผลลัพธ์ที่ส่งมอบ:

- โรงเรียนมีข้อมูลเยี่ยมบ้านครบถ้วน
- ติดตามนักเรียนที่ยังไม่ได้เยี่ยมบ้านได้
- วิเคราะห์ความเสี่ยงรายห้องและรายชั้นได้

### Phase 4: ธนาคารขยะ

เป้าหมาย: แปลงขยะพลาสติกเป็นเงินและสะสมยอดให้นักเรียน

ขอบเขต:

- ตั้งค่าประเภทขยะและราคาต่อกิโลกรัม
- บันทึกน้ำหนักจากการชั่ง
- คำนวณยอดเงินอัตโนมัติ
- Student Portal แสดงยอดเงินและประวัติ
- รายงานปริมาณขยะและยอดเงินสะสม

ผลลัพธ์ที่ส่งมอบ:

- นักเรียนเห็นยอดธนาคารขยะของตนเอง
- โรงเรียนรายงานปริมาณขยะและมูลค่าได้
- ปรับราคาใหม่ได้โดยไม่กระทบรายการย้อนหลัง

### Phase 5: รายงานผู้บริหารและการอนุมัติ

เป้าหมาย: ทำให้ข้อมูลใช้บริหารจริงและควบคุมคุณภาพข้อมูล

ขอบเขต:

- workflow อนุมัติรายการสำคัญ
- รายงานผู้บริหารแบบ drill-down
- export PDF
- ตัวกรองปีการศึกษา ชั้น ห้อง และช่วงเวลา
- dashboard นักเรียนกลุ่มเสี่ยง

ผลลัพธ์ที่ส่งมอบ:

- ผู้บริหารเห็นภาพรวมโรงเรียน
- ฝ่ายปกครองใช้ข้อมูลเพื่อติดตามนักเรียนรายคนได้
- ข้อมูลสำคัญมีสถานะตรวจสอบ/อนุมัติ

## ลำดับที่แนะนำ

ควรเริ่ม Phase 1 และ Phase 2 ก่อน เพราะใช้ข้อมูลนักเรียนและ Student Portal เดิมได้ทันที จากนั้นทำ Phase 3 เพราะเกี่ยวข้องกับข้อมูลอ่อนไหวและฟอร์มค่อนข้างใหญ่ แล้วจึงต่อ Phase 4 และ Phase 5 เพื่อเพิ่มคุณค่าด้านสิ่งแวดล้อมและรายงานผู้บริหาร

## Slice แรกที่ควรเริ่มเขียนโค้ด

1. เพิ่ม field toggle ใน `SystemSetting`
2. เพิ่ม ViewModel และ UI toggle ใน `Settings`
3. เพิ่ม entity `StudentCareProfile`, `BehaviorScoreTransaction`, `GoodnessBankTransaction`
4. เพิ่มหน้า Student Portal สำหรับแสดงคะแนน
5. เพิ่มหน้า Teacher Area สำหรับบันทึกคะแนนรายคน

slice นี้เล็กพอสำหรับทดสอบเร็ว แต่เป็นแกนกลางที่ทำให้โมดูลอื่นต่อยอดได้ทั้งหมด
## Phase 7: แฟ้มดูแลนักเรียนรายบุคคลและการพิมพ์

เป้าหมาย: รวมข้อมูลดูแลนักเรียนทุกมิติไว้ในแฟ้มรายคนสำหรับครูที่ปรึกษาและผู้บริหาร

ขอบเขต:

- หน้าแฟ้มรายบุคคลจากฝั่งครูและผู้บริหาร
- สรุปคะแนนพฤติกรรม ธนาคารความดี ธนาคารขยะ และสถานะความเสี่ยง
- ข้อมูลผู้ปกครอง รายได้ รูปผู้ปกครอง ข้อมูลบ้าน และรูปจากการเยี่ยมบ้าน
- ประวัติเยี่ยมบ้านพร้อมปัญหาที่พบ พฤติกรรมเสี่ยง ข้อสังเกตครู และแผนช่วยเหลือ
- ลิงก์เปิดพิกัดบ้านในแผนที่เมื่อมีการแชร์โลเคชั่น
- หน้า Print สำหรับพิมพ์หรือบันทึกเป็น PDF จากเบราว์เซอร์

ผลลัพธ์ที่ส่งมอบ:

- ครูที่ปรึกษาเปิดแฟ้มจากตารางดูแลนักเรียนและรายงานได้
- ผู้บริหารเปิดแฟ้มจากรายงานระบบดูแลนักเรียนได้
- ใช้เป็นหลักฐานประกอบการประชุม ติดตามช่วยเหลือรายกรณี และจัดทำเอกสารรายบุคคลได้
## Phase 8: ระบบติดตามช่วยเหลือรายกรณี

เป้าหมาย: เปลี่ยนข้อมูลความเสี่ยงและแผนช่วยเหลือให้เป็นงานติดตามที่มีเจ้าของ สถานะ และวันครบกำหนด

ขอบเขต:

- สร้างเคสติดตามช่วยเหลือจากแฟ้มดูแลนักเรียนรายบุคคล
- ระบุประเด็นที่ต้องติดตาม แผนช่วยเหลือ ความเร่งด่วน และวันนัดติดตาม
- อัปเดตสถานะเคสเป็น เปิดเคส กำลังติดตาม เสร็จแล้ว หรือยกเลิก
- บันทึกผลติดตามล่าสุดและวันที่ปิดเคส
- แสดงเคสในแฟ้มรายบุคคลและหน้า Print
- รายงานครูแสดงเคสที่ยังเปิดอยู่ เรียงตามความเร่งด่วนและวันครบกำหนด พร้อมแสดงรายการเกินกำหนด

ผลลัพธ์ที่ส่งมอบ:

- ครูที่ปรึกษาติดตามเด็กกลุ่มเสี่ยงแบบเป็นงานต่อเนื่องได้
- หัวหน้าระดับหรือผู้บริหารตรวจดูความคืบหน้าในแฟ้มรายคนได้
- โรงเรียนเห็นงานช่วยเหลือที่ค้างอยู่และเคสที่เกินกำหนดได้ชัดเจน
## Phase 9: แจ้งเตือนงานติดตามบนแดชบอร์ดครู

เป้าหมาย: ทำให้ครูที่ปรึกษาเห็นงานติดตามช่วยเหลือที่ต้องรีบจัดการทันทีเมื่อเปิดหน้าดูแลนักเรียน

ขอบเขต:

- แสดงจำนวนเคสติดตามที่ยังเปิดอยู่
- แยกจำนวนเคสที่เกินกำหนดและครบกำหนดวันนี้
- แสดงรายการเคสที่เกินกำหนด ครบกำหนดวันนี้ หรือใกล้ถึงกำหนดใน 7 วัน
- เรียงรายการตามวันนัดติดตาม ความเร่งด่วน และวันที่เปิดเคส
- ลิงก์จากรายการแจ้งเตือนไปยังแฟ้มดูแลนักเรียนรายบุคคลเพื่ออัปเดตสถานะหรือผลติดตาม

ผลลัพธ์ที่ส่งมอบ:

- ครูเห็นงานติดตามสำคัญบนหน้าแรกของโมดูลดูแลนักเรียน
- เคสเกินกำหนดและครบกำหนดวันนี้ถูกเน้นให้จัดการก่อน
- ใช้ข้อมูล `StudentCareFollowUpCase` เดิมได้โดยไม่ต้องเพิ่มโครงสร้างฐานข้อมูล
